# 五层架构

Chaos 引擎采用分层架构设计，作为一套独立的**世界模拟与涌现逻辑层**运行于传统游戏引擎之上。

---

## 架构概览

```
┌─────────────────────────────────────────────────────────────┐
│                    第五层：宏观趋势层                        │
│         世界趋势参数、历史演变、共业驱动                      │
├─────────────────────────────────────────────────────────────┤
│                    第四层：资产记录层                        │
│         知识资产 NFT、历史哈希、数字产权                      │
├─────────────────────────────────────────────────────────────┤
│                    第三层：涌现生成层                        │
│         事件模板、上下文填充、约束求解                        │
├─────────────────────────────────────────────────────────────┤
│                    第二层：共识场层                          │
│         应力计算、阈值监控、贡献者图谱                        │
├─────────────────────────────────────────────────────────────┤
│                    第一层：基础交互层                        │
│         移动、战斗、交易、采集、聊天                          │
└─────────────────────────────────────────────────────────────┘
```

---

## 第一层：基础交互层

提供玩家与游戏世界交互的基本接口。

**核心功能**：
- 移动控制
- 物品使用
- 技能释放
- 交易操作
- 聊天通信

**输出**：所有有效操作转化为**行为事件**（Action Events）向上层传递。

---

## 第二层：共识场层

Chaos 引擎的核心层。负责接收、聚合、解释所有玩家的行为事件。

**核心概念**：
- **社会应力**：玩家行为对不同维度的贡献值
- **应力分布图**：覆盖整个游戏世界的连续函数
- **涌现阈值**：触发上层事件的临界值

**应力类型**：

| 应力类型 | 描述 | 数据范围 |
|:---|:---|:---|
| S_combat | 战斗烈度 | [-1.0, 1.0] |
| S_trade | 贸易活跃度 | [-1.0, 1.0] |
| S_ecology | 生态平衡 | [-1.0, 1.0] |
| S_order | 秩序/混乱 | [-1.0, 1.0] |

---

## 第三层：涌现生成层

当共识场应力突破阈值时被触发，负责生成具体的涌现事件。

**生成流程**：
1. 事件模板筛选
2. 上下文参数填充
3. 约束求解与冲突检测
4. 执行与广播

**涌现类型**：
- **微涌现**：影响单个玩家或小队
- **局部涌现**：影响一个城镇或一片区域
- **区域涌现**：影响多个相连区域
- **世界级涌现**：影响整个游戏世界

---

## 第四层：资产记录层

基于分布式账本技术，记录游戏中具有稀缺性的知识资产。

**记录内容**：
- 技法与连招蓝图
- 制造配方
- 重要世界历史事件哈希
- 玩家发现者署名

**核心特性**：不可篡改、可追溯、全球唯一。

---

## 第五层：宏观趋势层

维护一组缓慢变化的世界趋势参数，为世界提供长周期的"气候"变化。

**趋势参数示例**：

| 参数 | 范围 | 影响 |
|:---|:---|:---|
| 王朝气数 | [0, 1] | S_order 阈值、NPC 守卫强度 |
| 灵气潮汐 | [-1, 1] | 魔法效果强度、魔力生物频率 |
| 格物致知 | [-1, 1] | 新配方发现概率 |

**驱动机制**：由全体玩家的集体行为积分驱动，反向调制下层共识场的应力计算规则。
