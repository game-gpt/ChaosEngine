# 数据流

Chaos 引擎的数据流设计确保玩家行为的积累是连续过程，而世界的显著变化是离散事件。

---

## 核心数据流

```
┌──────────────┐
│   玩家操作   │
└──────┬───────┘
       │
       ▼
┌──────────────────────────────────────────────────────────┐
│                    基础交互层                             │
│  输出：行为事件（玩家ID、坐标、动作类型、目标）            │
└──────────────────────────┬───────────────────────────────┘
                           │
                           ▼
┌──────────────────────────────────────────────────────────┐
│                    共识场层                               │
│  处理：更新对应区域的应力值 & 贡献者图谱                   │
│                                                           │
│  ┌─────────────────────┐    ┌─────────────────────┐      │
│  │  应力未达阈值       │    │  应力达到阈值       │      │
│  │  仅记录，无显著变化  │    │  触发信号传递       │      │
│  └─────────────────────┘    └──────────┬──────────┘      │
└─────────────────────────────────────────┼────────────────┘
                                          │
                                          ▼
┌──────────────────────────────────────────────────────────┐
│                    涌现生成层                             │
│  处理：根据上下文执行涌现规则，生成具体游戏内容            │
│                                                           │
│  ┌─────────────────────┐    ┌─────────────────────┐      │
│  │  生成可交互实体     │    │  生成知识资产       │      │
│  │  返回基础交互层     │    │  提交资产记录层     │      │
│  └─────────────────────┘    └─────────────────────┘      │
└──────────────────────────────────────────────────────────┘
                           │
                           ▼
┌──────────────────────────────────────────────────────────┐
│                    资产记录层                             │
│  处理：存证知识资产、记录历史事件哈希                      │
└──────────────────────────────────────────────────────────┘
                           │
                           ▼
┌──────────────────────────────────────────────────────────┐
│                    宏观趋势层                             │
│  处理：定期根据全球行为统计更新世界趋势参数                │
│  反馈：趋势参数反向调制共识场的应力计算权重                │
└──────────────────────────────────────────────────────────┘
```

---

## 关键数据流路径

### 路径一：行为事件 → 共识场更新

```
玩家击杀怪物
    │
    ▼
行为事件: { player_id, action: "kill", target: "goblin", location: (x, y, z) }
    │
    ▼
共识场更新:
  - S_combat += 0.0001 (战斗烈度)
  - S_ecology += 0.0001 (生态破坏)
  - 贡献者图谱记录玩家贡献
```

### 路径二：阈值触发 → 涌现生成

```
应力值突破阈值 (S_ecology > 0.8)
    │
    ▼
触发信号: { location, dimension: "S_ecology", value: 0.82, contributors: [...] }
    │
    ▼
涌现生成:
  - 筛选事件模板: ["野火", "兽潮", "树人觉醒"]
  - 上下文填充: 位置、规模、参与者倾向
  - 约束求解: 检查与当前世界状态的冲突
  - 执行: 生成 NPC、改变地形、触发任务
```

### 路径三：资产记录 → 宏观趋势

```
玩家发现新技法
    │
    ▼
资产记录:
  - 生成技法 NFT
  - 记录发现者署名
  - 写入历史链
    │
    ▼
宏观趋势更新:
  - 格物致知参数上升
  - 反向调制共识场的新配方发现概率
```

---

## 数据流优化策略

| 策略 | 描述 |
|:---|:---|
| 降频计算 | 共识场应力以 30 秒至 1 分钟间隔更新 |
| 批量聚合 | 按类型聚合行为事件，一次性计算应力增量 |
| 稀疏存储 | 仅存储应力值显著偏离 0 的区域 |
| 客户端预测 | 非关键交互采用客户端预测 + 服务器异步验证 |

---

## 核心优势

**连续积累，离散触发**：服务器无需高频计算整个世界状态，仅在涌现触发时进行集中计算，极大降低计算负载，支撑大规模同时在线。
